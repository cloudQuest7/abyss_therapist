COMPOSITE INDEX CREATION GUIDE
===============================

WHAT ARE INDEXES?
==================
Indexes are like search tables - they let Firestore quickly find documents when you filter AND sort at the same time.

Error Message Means:
"The query requires an index" = You're searching by one field AND sorting by another = Need an index


QUICKEST WAY - Using Firebase Console Links
==============================================

1. Open your browser console (F12 in Chrome)
2. Look at the error messages
3. You'll see links like:
   "You can create it here: https://console.firebase.google.com/v1/r/project/abyss-1d41c/firestore/index..."

4. CLICK THOSE LINKS - Firebase will auto-create the index
5. You'll see a loading bar → "Index created"
6. Done!


MANUAL WAY - If links don't work
==================================

STEP 1: Open Firebase Console
- Go to https://console.firebase.google.com
- Select your project (abyss-1d41c)
- Click "Firestore Database" in left menu

STEP 2: Go to Indexes Tab
- You should see tabs at top: "Data", "Rules", "Indexes"
- Click "Indexes"

STEP 3: Create Index #1 (community-posts)
- Click "Create Index" button
- Collection dropdown: Select "community-posts"
- Add first field:
  * Field name: userId
  * Type: Ascending (▲)
- Add second field (click + Add field):
  * Field name: timestamp
  * Type: Descending (▼)
- Collection group: Leave unchecked
- Click "Create Index"
- Wait for "Enabled" status (usually 1-2 min)

STEP 4: Create Index #2 (chats)

WHAT TO LOOK FOR
=================

✅ STATUS SHOULD SAY "Enabled" (green)
❌ If it says "Building..." - wait 1-2 minutes
❌ If it says something red - check field names match exactly

FIELD NAMES MUST MATCH EXACTLY:
- userId (with capital I)
- timestamp (lowercase t)
- createdAt (with capital A)


AFTER ALL INDEXES CREATED
===========================

1. Go back to your app
2. Hard refresh: Ctrl+Shift+R (or Cmd+Shift+R on Mac)
3. Check browser console (F12)
4. You should NOT see "requires an index" errors anymore
5. Analytics should load with real-time data!
